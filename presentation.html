✅ BCNF Normalization

All tables satisfy Boyce-Codd Normal Form (BCNF):

1NF ✓

  • All attributes are atomic
  • Each column has a single value
  • No repeating groups

2NF ✓

  • Is in 1NF
  • No partial dependencies
  • Non-key attributes depend on full PK

3NF ✓

  • Is in 2NF
  • No transitive dependencies
  • Non-key depends only on PK

BCNF ✓

  • Is in 3NF
  • Every determinant is a candidate key
  • No anomalies possible

⚡ Database Trigger — Prevent Overbooking

CREATE TRIGGER trg_prevent_overbooking
BEFORE INSERT ON REGISTRATION
BEGIN
    SELECT CASE
        WHEN (
            SELECT COUNT(*) FROM REGISTRATION
            WHERE Event_ID = NEW.Event_ID
        ) >= (
            SELECT Max_Seats FROM EVENT
            WHERE Event_ID = NEW.Event_ID
        )
        THEN RAISE(ABORT, 'EVENT_FULL')
    END;
END;

🔒 Automatically prevents registration when an event reaches maximum capacity. Fires before every INSERT on REGISTRATION table.

👁️ Database VIEW — Participation Summary

CREATE VIEW vw_event_participation AS
SELECT
    E.Event_ID,
    E.Name       AS Event_Name,
    E.Date       AS Event_Date,
    E.Venue,
    E.Max_Seats,
    F.Name       AS Faculty_Name,
    COUNT(R.Reg_ID) AS Participant_Count
FROM EVENT E
LEFT JOIN FACULTY F ON E.Faculty_ID = F.Faculty_ID
LEFT JOIN REGISTRATION R ON E.Event_ID = R.Event_ID
GROUP BY E.Event_ID;

📊 Uses JOIN across 3 tables + GROUP BY aggregation. Provides a real-time participation dashboard for admins.

🔄 Transaction — Event Registration

BEGIN TRANSACTION;

-- Step 1: Verify event exists
SELECT Event_ID, Name, Max_Seats
FROM EVENT WHERE Event_ID = ?;

-- Step 2: Insert registration
-- Trigger fires here → checks capacity
INSERT INTO REGISTRATION (Student_ID, Event_ID)
VALUES (?, ?);

COMMIT;  -- Success
-- OR
ROLLBACK; -- On error (duplicate/full)

🔐 ACID-compliant transaction ensures atomicity. Either registration succeeds fully or rolls back completely.
